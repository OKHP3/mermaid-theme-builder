MISSION

Update Mermaid Theme Builder so its supported built-in themes and brand presets are directly aligned with the three public brand ecosystems above:

- OverKill Hill P³
- AskJamie
- Glee-fully Personalizable Tools

The purpose is not merely to add three color palettes. The purpose is to make Mermaid Theme Builder capable of generating reusable Mermaid visual governance artifacts that reflect each brand’s look, feel, typography, tone, color system, and diagram-use posture.

Keep the product thesis intact:

Mermaid Theme Builder is a static, browser-based visual governance utility for Mermaid diagrams. It helps users define a theme once, apply it to existing Mermaid code, preview the styled result, copy/export the result, and generate reusable prompt scaffolds so LLMs stop hallucinating diagram styles.

STRICT CONSTRAINTS

Do not add:
- backend
- login
- database
- AI API calls
- payment
- file upload
- user accounts
- cloud storage

Do not fork Mermaid.
Do not copy Mermaid source into this repo.
Continue using Mermaid as an npm dependency.

Keep the app local-only. User Mermaid input must remain in the browser.

Do not turn this into a Mermaid.ai clone.

OBJECTIVE 4 — ATTRIBUTION / THEME FOOTNOTE SUPPORT

Add optional theme attribution support.

There should be two attribution layers:

Layer 1 — Source metadata comments:
Always available and preferably enabled by default in exports.

Generated Mermaid should be able to include source comments like:

%% Created with Mermaid Theme Builder
%% Tool: https://overkillhill.com/projects/mermaid-theme-builder/
%% Theme: OverKill Hill P³
%% Theme Version: 0.1.0
%% Created: ISO_TIMESTAMP
%% Theme ID: overkill-hill

Layer 2 — Visible attribution badge:
Optional, default ON if reasonable, user-toggleable.

For flowchart-style diagrams only, add a small visible badge node when enabled.

The badge should say something concise, such as:
Created with Mermaid Theme Builder · Theme: OverKill Hill P³

The badge should be styled discreetly using the selected theme’s muted/border colors.

If Mermaid click-link syntax is safe for the diagram type, make the badge link to:
https://overkillhill.com/projects/mermaid-theme-builder/

If the diagram type does not safely support a visible linked badge, fall back to metadata comments only.

Do not corrupt the diagram layout. Attribution is best-effort and must be optional.

Add UI controls:
- Include metadata comments: on/off
- Include visible attribution badge: on/off
- Theme name/title field for custom user-created themes
- Theme version field or generated version default
- Theme created/updated timestamp display or metadata
