# REPLIT: Repository Cleanup and Structure Rationalization

## PROBLEM

The repository is a pnpm monorepo with workspace scaffolding that was generated by Replit's default full-stack template. The Mermaid Theme Builder is a 100% client-side static tool with no backend, no API, no database, and no storage. The monorepo structure ships dead weight that contradicts the product's architecture, confuses context for AI agents, bloats `node_modules`, and makes the repo look like enterprise middleware instead of a focused utility.

**Current structure (what exists):**

```
/                              ← Root monorepo shell
├── artifacts/                 ← Workspace container
│   ├── mermaid-theme-builder/ ← THE ACTUAL PRODUCT (this is the only thing that matters)
│   │   ├── src/
│   │   │   ├── components/
│   │   │   │   ├── ui/        ← 60+ UNUSED shadcn components (dead code)
│   │   │   │   ├── ThemeBuilder.tsx
│   │   │   │   ├── MermaidPreview.tsx
│   │   │   │   ├── ColorSwatch.tsx

│   │   │   │   ├── ClassBrowser.tsx

**What's dead and why:**

| Path | Why It's Dead | Evidence |
|------|--------------|---------|
| `artifacts/api-server/` | Plan says "no backend" (TR-001, NFR). Tool is 100% client-side. | Express 5, Pino logging, cookie parser, CORS, health endpoint |
| `artifacts/mockup-sandbox/` | Not in any plan document. Replit template artifact. | Component preview server with chokidar watcher |
| `lib/db/` | Plan says "no storage" (NFR). No database exists or is planned. | PostgreSQL + Drizzle ORM with migration tooling |
| `lib/api-spec/` | No API to specify. | OpenAPI spec + Orval codegen |
| `lib/api-zod/` | No API to validate. | Zod schemas generated from OpenAPI |
| `lib/api-client-react/` | No API to call. | React Query hooks with auth token support |
| `src/components/ui/` (60+ files) | Zero imports from the product code. Every UI component is custom-built. | Accordion, Calendar, Carousel, Chart, Command, ContextMenu, Drawer, Form, etc. |
| `replit.md` | Auto-generated by Replit. Not project documentation. | Replit template boilerplate |

The target structure eliminates all dead packages, removes the monorepo wrapper, and places the Mermaid Theme Builder at the repo root.

**Target structure:**

```

## EXECUTION STEPS

### Step 1: Inventory before deletion

Before deleting anything, run these commands and review the output:

```bash
# List all workspace packages
cat pnpm-workspace.yaml

# Check what's in each dead package
ls -la artifacts/api-server/ 2>/dev/null
ls -la artifacts/mockup-sandbox/ 2>/dev/null
ls -la lib/db/ 2>/dev/null
ls -la lib/api-spec/ 2>/dev/null
ls -la lib/api-zod/ 2>/dev/null
ls -la lib/api-client-react/ 2>/dev/null

# Count unused shadcn components
ls artifacts/mermaid-theme-builder/src/components/ui/ 2>/dev/null | wc -l

# Verify no imports from dead packages
grep -r "api-server\|mockup-sandbox\|api-client\|api-spec\|api-zod\|@workspace/db" artifacts/mermaid-theme-builder/src/ 2>/dev/null
```

If the grep returns zero results (expected), the dead packages have no imports from the product code and are safe to delete.

### Step 2: Verify shadcn usage

```bash
# Check which shadcn components are actually imported
grep -r "from.*components/ui/" artifacts/mermaid-theme-builder/src/ --include="*.tsx" --include="*.ts" 2>/dev/null | grep -v "node_modules" | sort -u
```

Any component NOT listed in the grep output is dead code. Remove the entire `ui/` directory if zero imports are found.

If some shadcn components ARE imported, keep only those specific files and delete the rest.

### Step 3: Copy the product to root

```bash
# Create a temporary backup
cp -r artifacts/mermaid-theme-builder /tmp/mtb-backup

# Copy all product source files to repo root
cp -r artifacts/mermaid-theme-builder/* .
cp artifacts/mermaid-theme-builder/.* . 2>/dev/null  # hidden files
```

### Step 4: Delete dead packages

```bash
# Remove dead workspace packages
rm -rf artifacts/api-server/
rm -rf artifacts/mockup-sandbox/
rm -rf lib/api-client-react/
rm -rf lib/api-spec/
rm -rf lib/api-zod/
rm -rf lib/db/

# Remove the now-empty workspace containers
rm -rf artifacts/
rm -rf lib/

# Remove monorepo config
rm -f pnpm-workspace.yaml
rm -f tsconfig.base.json
rm -f replit.md
```

### Step 5: Update package.json

The root `package.json` should be the product's package.json, not the monorepo wrapper. Replace the root package.json with the one from `artifacts/mermaid-theme-builder/package.json`. Ensure it:

- Has NO `workspaces` field
- Has `name: "mermaid-theme-builder"`
- Has `scripts.dev`, `scripts.build`, `scripts.preview`
- Pins `mermaid: "^11.14.0"`
- Removes any references to workspace packages (`@workspace/*`)

### Step 6: Update vite.config.ts

Ensure `base` is set for GitHub Pages:

```ts
export default defineConfig({
  base: '/mermaid-theme-builder/',
  // ... rest of config
});
```

### Step 7: Remove unused shadcn components

If Step 2 confirmed zero shadcn imports:

```bash
rm -rf src/components/ui/
```

If some are used, remove only the unused ones. The product uses custom components (ThemeBuilder, MermaidPreview, ColorSwatch, WarningBanner, ClassBrowser, CapabilityNote, DiagramInventory, PromptScaffoldModal). These are NOT shadcn and must be kept.

### Step 8: Clean install

```bash
# Remove old lock file and node_modules
rm -rf node_modules/
rm -f pnpm-lock.yaml

# Fresh install from the flat package.json
pnpm install

# Verify build
pnpm build

# Verify dev server
pnpm dev
```

### Step 9: Verify the app works

1. Open the dev server in browser
2. Paste a flowchart diagram
3. Select a palette
4. Verify preview renders
5. Verify all 3 export buttons work
6. Verify all 7+ palettes are selectable

### Step 10: Commit the cleanup

```bash
git add -A
git commit -m "chore: flatten monorepo, remove dead packages (api-server, db, mockup-sandbox, unused shadcn)

BREAKING: Repository structure changed from pnpm monorepo to flat Vite project.
The app source is now at repo root instead of artifacts/mermaid-theme-builder/.

Removed:
- artifacts/api-server/ (Express backend, never used)
- artifacts/mockup-sandbox/ (component preview, never used)
- lib/db/ (PostgreSQL + Drizzle ORM, never used)
- lib/api-spec/ (OpenAPI spec, never used)
- lib/api-zod/ (Zod schemas, never used)
- lib/api-client-react/ (React Query hooks, never used)
- src/components/ui/ (60+ unused shadcn components)
- pnpm-workspace.yaml (monorepo config)
- tsconfig.base.json (shared TS config)
- replit.md (Replit auto-docs)

The Mermaid Theme Builder is a static client-side tool.
It has no backend, no API, no database, and no storage.
These packages were Replit template scaffolding that contradicted the product architecture."

git push origin main
```
