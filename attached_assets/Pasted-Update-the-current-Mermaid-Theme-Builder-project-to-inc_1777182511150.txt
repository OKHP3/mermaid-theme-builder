Primary objective:
Add a productized Diagram Inventory / Capability Registry layer to Mermaid Theme Builder.

This should help users answer:
- What kind of diagram am I making?
- Does Mermaid support it?
- Is it native, partial, emulatable, unsupported, beta, or external?
- How safely can Mermaid Theme Builder theme it?
- What example should I start from?
- What warnings apply before I trust the output?

Maintain the existing core workflow:
paste Mermaid → detect diagram family → select/edit theme → generate themed Mermaid → preview → copy/export

Do not add:
- backend
- login
- payment
- database
- AI API calls
- file upload
- cloud storage
- analytics capture of pasted diagram content
- user accounts

## Task 4 — Add support warnings

For each registry entry, include a clear warning where applicable.

Examples:

BPMN 2.0:
“Mermaid flowcharts can approximate BPMN-like visual flows, but they do not implement BPMN 2.0 semantics.”

C4:
“Mermaid C4 support may vary by renderer and version. Treat as partial unless verified.”

Architecture:
“Architecture diagrams may be beta or renderer-dependent depending on Mermaid version.”

Sankey / XY / Block / Packet / Radar / Treemap:
“Beta or renderer-dependent. Validate in target renderer before publication.”

Sequence:
“Supported natively, but styling surface is narrower than flowchart; classDef/linkStyle patterns may not apply.”

Flowchart:
“Best-supported family for rich theme styling, classDef, subgraphs, links, and visual governance.”

## Task 5 — Integrate with diagram detection

When a user pastes Mermaid code, detection should connect to the registry.

The UI should show:
- detected diagram type,
- support status,
- theme confidence,
- warnings,
- related example,
- whether Mermaid Theme Builder can safely apply rich styling or only generic theme variables.

Do not overclaim support.

If the diagram type is unknown:
- mark Support Status as Unknown / Gap,
- show a friendly warning,
- still allow generic theming if safe.

## Acceptance criteria

The update is complete when:

1. A Diagram Inventory / Capability Registry section exists in the app.
2. Native Mermaid diagram families are represented.
3. Unsupported/emulated diagram families are represented honestly.
4. At least one example or placeholder exists for each tracked Mermaid family.
5. Pasted diagram detection references the registry.
6. Warnings are visible for beta, partial, emulated, unsupported, and external families.
7. README/docs explain the distinction between taxonomy and capability registry.
8. No BFS/employer references are introduced.
9. No third-party company brand presets are introduced.
10. Existing paste → theme → preview → copy/export workflow still works.

At the end, report:
- files added,
- files changed,
- registry entries added,
- examples wired,
- warnings added,
- build/test status,
- remaining gaps before V0.3 public alpha.
