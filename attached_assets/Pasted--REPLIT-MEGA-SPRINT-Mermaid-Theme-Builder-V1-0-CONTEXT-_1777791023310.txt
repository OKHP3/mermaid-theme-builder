# REPLIT MEGA SPRINT: Mermaid Theme Builder V1.0

## CONTEXT

You are working on the Mermaid Theme Builder, a browser-based visual governance tool for Mermaid diagrams. The app lives at `artifacts/mermaid-theme-builder/`. It is built with React + TypeScript + Vite + Tailwind CSS and uses Mermaid.js ^11.14.0. It deploys to GitHub Pages at okhp3.github.io/mermaid-theme-builder/.

**Read AGENTS.md before you start.** It contains brand separation rules that are non-negotiable. Never use BFS/Builders FirstSource hex values (#00205B, #003087, #002F86, #B3C1DB, #D6E5F9, #D0D0CE, #C8102E). This is a personal OverKill Hill P3 project.

This sprint is a comprehensive build-out covering every planned improvement. Execute each phase sequentially. Each phase should compile and render before moving to the next. Commit to GitHub after each phase completes successfully.

-----

## PHASE 1: UX RESTRUCTURING (4 Tabbed Sub-Screens)

### Problem

The current single-screen layout puts every feature on one page: palette selector, color editor, code input, preview, ClassBrowser, DiagramInventory, CapabilityNote, PromptScaffoldModal, and export buttons. This fails on mobile and creates cognitive overload on desktop. There is no workflow progression.

### Solution

Break the single screen into 4 tabbed sub-screens. Each tab has one job.

### Task 1.1: Create Tab Navigation Component

Create `TabNavigation.tsx`:

```tsx
type AppTab = "apply" | "compose" | "examples" | "reference";
```

Requirements:

- Horizontal tab bar at top on desktop (>=768px)
- Bottom navigation bar on mobile (<768px), fixed position
- Active tab has clear visual indicator using the current theme’s accent color
- Tab labels: “Apply” (default), “Compose”, “Examples”, “Reference”
- Tab icons (use Lucide or inline SVGs): Apply = wand/magic, Compose = palette, Examples = grid/gallery, Reference = book/info
- Smooth transitions between tabs (no page reload, React state only)
- URL hash routing optional but nice to have (#apply, #compose, #examples, #reference)

- Preview panel: rendered Mermaid SVG. Show spinner while rendering. Show error message inline if Mermaid throws.
- Original/Themed toggle: lets user compare before/after without re-pasting
- Warnings strip: only visible if warnings exist (existing init directives detected, non-printable characters, excessively long labels)
- Export buttons:
  - “Styled Code” = copy styled Mermaid code with init directive to clipboard
  - “Markdown” = copy markdown bootstrap block to clipboard
  - “Prompt” = copy AI prompt scaffold to clipboard
  - “SVG” = download the rendered SVG as a file (NEW feature)
- Toast notification on copy (“Copied to clipboard!”)
