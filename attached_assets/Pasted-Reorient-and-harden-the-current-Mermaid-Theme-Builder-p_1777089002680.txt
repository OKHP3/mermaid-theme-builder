PART 4 — Mermaid Capability Registry

Create something like:
- `src/data/mermaid-capabilities.json`
or
- `src/lib/diagramRegistry.ts`

The registry should distinguish:
1. Diagram types Mermaid can probably render.
2. Diagram types the app can detect.
3. Diagram types the app can theme confidently.
4. Diagram types that are beta/experimental/generic-theme-only.

Use Mermaid 11.14.x as the reviewed baseline unless the installed dependency is newer.

Include at least:

PART 5 — Theme metadata and attribution

Add Theme Metadata & Attribution support.

Requirements:
1. Allow the user to name a custom theme.
2. Track:
   - themeName
   - themeId
   - createdAt
   - updatedAt
   - createdWith: Mermaid Theme Builder
   - createdBy: OverKill Hill P³ / Jamie Hill
   - toolVersion
   - toolUrl
   - sourceTheme
3. Include metadata in exported theme packages and Markdown bootstraps.
4. Include Mermaid source comments by default:
   - Theme
   - Created with
   - Tool URL
   - Tool version
   - Created/updated timestamp
5. Add toggles:
   - Include source metadata comments: default ON
   - Include visible attribution when supported: default OFF or “compatible diagrams only”
6. For flowchart/graph diagrams only, optionally inject a small attribution node.
7. If visible attribution is enabled and clickable links are requested, warn the user that Mermaid click behavior may require `securityLevel: "loose"` and may not work in all renderers.
8. Do not inject visible attribution into sequence, timeline, mindmap, gantt, ER, class, state, or beta diagrams in V1.
9. Do not break Mermaid syntax.
10. Visible attribution must be user-controllable.

Suggested source metadata comments:

%% Theme: {themeName}
%% Theme ID: {themeId}
%% Created with: Mermaid Theme Builder by OverKill Hill P³
%% Tool URL: https://overkillhill.com/projects/mermaid-theme-builder/
%% Tool Version: {toolVersion}
%% Theme Created: {createdAt}
