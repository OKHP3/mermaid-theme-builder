# REPLIT: V0.4 Release — Tabbed UI + Mobile Responsive

## OBJECTIVE

Restructure the Mermaid Theme Builder from a single-screen layout into 4 tabbed sub-screens. Each tab has one job. The current product works but puts every feature on one page: palette selector, color editor, code input, preview, ClassBrowser, DiagramInventory, CapabilityNote, PromptScaffoldModal, and export buttons. This creates cognitive overload on desktop and is completely unusable on mobile.

After this release, a first-time user on the Apply tab can paste code and copy styled output in under 30 seconds. On mobile, the app is fully usable without horizontal scrolling.

**Read AGENTS.md before starting.** No BFS hex values. No employer references. This is a personal OverKill Hill P3 project.

---

## ARCHITECTURE: 4 Tabs

```
┌─────────┬──────────┬──────────┬───────────┐
│  Apply  │ Compose  │ Examples │ Reference │
│(default)│          │          │           │
└─────────┴──────────┴──────────┴───────────┘
```

| Tab | One-Sentence Job | What Lives Here |
|-----|-----------------|-----------------|
| **Apply** (default) | Paste, theme, preview, copy. | Code textarea, palette strip, preview, export buttons. Nothing else. |
| **Compose** | Define your visual identity. Export bootstraps. | Full color editor, font controls, theme name, watermark toggle, bootstrap/scaffold export. |
| **Examples** | Browse and load example diagrams. | Example list grouped by family, preview with current theme, "Load into Apply" button. |
| **Reference** | Learn what Mermaid supports. | ClassBrowser, CapabilityNote, DiagramInventory, support-level indicators. |

## STEP 1: Create Shared State (React Context)

Create a file `src/context/AppContext.tsx` (or add to existing state management):

```tsx
import { createContext, useContext, useState, ReactNode } from 'react';

type AppTab = 'apply' | 'compose' | 'examples' | 'reference';

interface AppState {
  // Navigation
  activeTab: AppTab;
  setActiveTab: (tab: AppTab) => void;

  // Theme (shared between Apply and Compose)
  selectedPaletteId: string;
  setSelectedPaletteId: (id: string) => void;
  customColors: Record<string, string>;
  setCustomColors: (colors: Record<string, string>) => void;
  fontFamily: string;
  setFontFamily: (font: string) => void;
  themeName: string;
  setThemeName: (name: string) => void;
  watermarkEnabled: boolean;
  setWatermarkEnabled: (enabled: boolean) => void;

  // Diagram (shared between Apply and Examples)
  inputCode: string;
  setInputCode: (code: string) => void;
  detectedFamily: string | null;

  // Color editor panel visibility
  colorEditorOpen: boolean;
  setColorEditorOpen: (open: boolean) => void;
}
```

**Critical behaviors:**
- When the user selects a palette in Compose, Apply reflects the change immediately.
- When the user clicks "Load into Apply" in Examples, the code pastes into Apply AND the active tab switches to Apply.
- When the user edits colors in the Apply slide-out panel, Compose reflects the changes.
- No prop drilling. All cross-tab state flows through context.

## STEP 2: Create Tab Navigation Component

Create `src/components/TabNavigation.tsx`:

### Desktop (>= 768px): Top tab bar

```
┌──────────────────────────────────────────────────────┐
│  ⚡ Apply    🎨 Compose    📋 Examples    📖 Reference │
└──────────────────────────────────────────────────────┘
```

- Horizontal bar at the top of the app, below any header/branding
- Each tab: icon + label
- Active tab: filled background using the current theme accent color, white text
- Inactive tabs: ghost/outline style, muted text
- Smooth transition (no page reload, just state change)
- Icons (use Lucide or inline SVG): Apply = Wand2, Compose = Palette, Examples = LayoutGrid, Reference = BookOpen

### Mobile (< 768px): Bottom navigation bar

```
┌───────────────────────┐
│                       │
│   (app content)       │
│                       │
├───────────────────────┤
│ ⚡  🎨  📋  📖       │  ← fixed bottom, icon + small label
└───────────────────────┘
```

- Fixed position at viewport bottom (like iOS/Android tab bars)
- 4 equally sized tap targets (minimum 48px height for touch)
- Icon above label, label text 11px
- Active tab: accent color icon + label. Inactive: muted gray.
- Content area gets `padding-bottom` to avoid being hidden behind the bar

### URL hash routing (nice to have, not required)
If feasible, update `window.location.hash` when tabs change (#apply, #compose, #examples, #reference) and read hash on load to restore tab state. This lets users bookmark or share a specific tab.

### Tablet layout (768-1023px): Stacked

Code input on top (40% height), preview below (60% height). Palette strip horizontal above code input. Export bar at bottom.

### Mobile layout (< 768px):

```
┌───────────────────────┐
│ Palette strip (scroll)│  ← horizontal scrollable, compact 28px swatches
├───────────────────────┤
│ Code input            │  ← 4 visible lines, tap to expand to 50% viewport
│ (collapsible)         │
├───────────────────────┤
│ Live Preview          │  ← takes remaining vertical space
│ (pinch-to-zoom)       │
├───────────────────────┤
│ [Edit][Code][MD][AI]  │  ← fixed bottom action bar (above tab nav)
└───────────────────────┘
```

### Palette Strip Component

Create `src/components/PaletteStrip.tsx`:

- Horizontal row of small circular color swatches (28px diameter each)
- Each swatch shows the palette's `primaryColor` as the fill
- Selected palette has a 2px ring/glow in the palette's accent color
- Palette name appears as a tooltip on hover (desktop) or below the strip (mobile)
- On mobile: horizontal scroll if palettes exceed viewport width
- Clicking a swatch sets `selectedPaletteId` in shared state
- This is NOT the full card grid with descriptions. That belongs in Compose.

### Color Editor Slide-Out

When user clicks "Edit Colors" button:

- **Desktop**: A 320px-wide panel slides in from the right edge. Contains all 13 color fields (reuse existing ColorSwatch components). Close button (X) at top. Clicking outside the panel also closes it.
- **Mobile**: Full-screen modal. All 13 color fields in a scrollable list. "Done" button at top to close. Swipe-down to dismiss.

The color editor is NOT permanently visible in Apply. It's on-demand only.

### What Does NOT Belong in Apply

These components must be removed from the Apply tab entirely:

- ❌ ClassBrowser (moves to Reference)
- ❌ DiagramInventory (moves to Examples + Reference)
- ❌ CapabilityNote (moves to Reference)
- ❌ PromptScaffoldModal trigger (Compose tab handles scaffold export)
- ❌ Full color editor (hidden behind "Edit Colors" button)
- ❌ Font family editor (moves to Compose)
- ❌ Theme name field (moves to Compose)
- ❌ Watermark toggle (moves to Compose)

### Export Buttons

Four buttons in the bottom bar:

1. **Copy Styled Code**: Copies the themed Mermaid code (with init directive) to clipboard. Toast: "Styled code copied!"
2. **Copy Markdown**: Copies the markdown bootstrap block. Toast: "Markdown bootstrap copied!"
3. **Copy Prompt**: Copies the AI prompt scaffold. Toast: "Prompt scaffold copied!"
4. **Edit Colors**: Opens the color editor slide-out/modal.

On mobile, use short labels or icons only: [🎨] [</>] [📝] [🤖]

### Toast Notifications

After any copy action, show a brief (2-second) toast notification at the bottom of the viewport confirming the action. Auto-dismiss. No manual close needed.

## STEP 8: Responsive Breakpoints

Define three breakpoints. Use Tailwind classes or CSS media queries:

| Breakpoint | Width | Tab Bar | Layout |
|-----------|-------|---------|--------|
| Mobile | < 768px | Bottom fixed nav | Single column, stacked |
| Tablet | 768-1023px | Top bar | Stacked with wider panels |
| Desktop | >= 1024px | Top bar | Side-by-side panels |

### Mobile-Specific Rules

1. **Tab bar**: Fixed at bottom, 56px height, 4 equal tap targets
2. **Content area**: Full-width, single column, scroll vertical
3. **Code input**: Collapsed to 4 visible lines by default. Tap to expand to 50% viewport. Tap "Done" or scroll past to collapse.
4. **Preview**: Takes remaining space between input and bottom bar
5. **Color editor**: Full-screen modal (not slide-out)
6. **Export buttons**: Fixed bottom action bar (positioned above the tab nav)
7. **No horizontal scrolling** anywhere except the palette strip
8. **Touch targets**: Minimum 44px for all interactive elements

### Desktop-Specific Rules

1. **Tab bar**: Top of content area, horizontal, text + icon labels
2. **Apply/Compose**: Two-panel side-by-side (left panel 40-45%, right panel 55-60%)
3. **Examples**: List panel (30%) + preview panel (70%)
4. **Reference**: Full-width single column (accordion list)
5. **Color editor**: 320px slide-out from right edge

## STEP 9: App Shell Update

Update `App.tsx` (or the root layout component) to use the new architecture:

```tsx
import { AppProvider } from './context/AppContext';
import { TabNavigation } from './components/TabNavigation';
import { ApplyTab } from './components/tabs/ApplyTab';
import { ComposeTab } from './components/tabs/ComposeTab';
import { ExamplesTab } from './components/tabs/ExamplesTab';
import { ReferenceTab } from './components/tabs/ReferenceTab';

function App() {
  return (
    <AppProvider>
      <div className="flex flex-col h-screen">
        {/* Desktop: tab bar at top */}
        <div className="hidden md:block">
          <TabNavigation position="top" />
        </div>

        {/* Tab content */}
        <main className="flex-1 overflow-auto pb-16 md:pb-0">
          <TabContent />
        </main>

        {/* Mobile: tab bar at bottom */}
        <div className="md:hidden fixed bottom-0 left-0 right-0 z-50">
          <TabNavigation position="bottom" />
        </div>
      </div>
    </AppProvider>
  );
}

function TabContent() {
  const { activeTab } = useAppContext();

  switch (activeTab) {
    case 'apply': return <ApplyTab />;
    case 'compose': return <ComposeTab />;
    case 'examples': return <ExamplesTab />;
    case 'reference': return <ReferenceTab />;
  }
}
```
