# PHASE 2: TABBED UI + MOBILE RESPONSIVE (V0.4)

## Step 2.1: Create Shared State

Create `src/context/AppContext.tsx`:

```tsx
import { createContext, useContext, useState, ReactNode } from 'react';

type AppTab = 'apply' | 'compose' | 'examples' | 'reference';

interface AppState {
  activeTab: AppTab;
  setActiveTab: (tab: AppTab) => void;
  selectedPaletteId: string;
  setSelectedPaletteId: (id: string) => void;
  customColors: Record<string, string>;
  setCustomColors: (colors: Record<string, string>) => void;
  fontFamily: string;
  setFontFamily: (font: string) => void;
  themeName: string;
  setThemeName: (name: string) => void;
  watermarkEnabled: boolean;
  setWatermarkEnabled: (enabled: boolean) => void;
  inputCode: string;
  setInputCode: (code: string) => void;
  colorEditorOpen: boolean;
  setColorEditorOpen: (open: boolean) => void;
  look: 'default' | 'neo' | 'handDrawn';
  setLook: (look: 'default' | 'neo' | 'handDrawn') => void;
}
```

Wrap the app in `<AppProvider>`. All tabs read from and write to this shared state. No prop drilling.

## Step 2.2: Create Tab Navigation

Create `src/components/TabNavigation.tsx`:

- **Desktop (>=768px)**: Horizontal bar at top. Labels: Apply, Compose, Examples, Reference. Icons optional (Lucide: Wand2, Palette, LayoutGrid, BookOpen).
- **Mobile (<768px)**: Fixed bottom bar, 56px height, 4 equal tap targets (minimum 44px), icon above 11px label.
- Active tab: filled accent color background. Inactive: muted/outline.
- Switching tabs changes `activeTab` in shared state. No page reload.

## Step 2.3: Build Apply Tab

Create `src/components/tabs/ApplyTab.tsx`:

**This tab contains ONLY:**
1. Compact palette strip (horizontal row of 28px circular swatches, selected has ring)
2. Code input textarea (monospace, auto-resize to max 50% viewport)
3. Live Mermaid preview (rendered SVG, spinner while rendering, error inline)
4. Original/Themed toggle
5. Export buttons: Copy Styled Code, Copy Markdown, Copy Prompt
6. "Edit Colors" button that opens a slide-out panel (desktop: 320px right side) or full-screen modal (mobile)
7. Warning banner (only if warnings exist)

**Apply does NOT contain:** ClassBrowser, DiagramInventory, CapabilityNote, PromptScaffoldModal, full color editor, font selector, theme name field, watermark toggle.

**Desktop**: Side-by-side (code left 45%, preview right 55%).
**Mobile**: Stacked. Palette strip horizontal scroll. Code input 4 visible lines (tap to expand). Preview takes remaining space. Export buttons fixed above bottom tab nav. No horizontal scrolling.

Toast notifications on copy (auto-dismiss after 2 seconds).

## Step 2.4: Build Compose Tab

Create `src/components/tabs/ComposeTab.tsx`:

Contains:
- Palette selector (full card style with descriptions, not compact strip)
- Full color editor (all 13 fields, always visible)

- Theme name text input (default: selected palette name)
- Watermark toggle (default ON)
- Look selector: Classic / Neo / Sketch (segmented control — see Phase 3 for details)
- Generic preview diagram (read-only, auto-generated sample with subgraph, decision, edges)
- Export buttons: Copy Bootstrap Markdown, Copy Prompt Scaffold
- Helper text: "Paste the bootstrap into your LLM before generating diagrams to enforce this visual standard."

**Desktop**: Left panel (controls 40%), right panel (preview 60%).
**Mobile**: Controls full-width, preview below fold, export fixed at bottom.

- Action buttons: "Load into Apply Tab" (sets inputCode + switches tab), "Copy Raw Code", "View Source" (toggles code block)

**Desktop**: List panel (30%) + preview panel (70%).
**Mobile**: List full-width, selecting example opens preview as bottom sheet.

# PHASE 3: MERMAID v11.14 FEATURE PARITY (V0.5)

## Step 3.1: Neo Look Selector

Mermaid v11.14.0 has three rendering styles: `default` (classic flat), `neo` (modern with drop shadows), `handDrawn` (sketch/rough). The Theme Builder currently generates only `default`.

### Add to shared state
Already included in Step 2.1: `look: 'default' | 'neo' | 'handDrawn'`

### Add Look Selector UI

In the **Compose tab**, add a segmented control below the palette selector:

```
┌──────────┬──────────┬──────────┐
│ Classic  │   Neo    │  Sketch  │
└──────────┴──────────┴──────────┘
```

- Classic = `'default'`. Label: "Classic". Description: "Clean flat rendering."
- Neo = `'neo'`. Label: "Neo". Description: "Modern with drop shadows. Requires Mermaid v11.14+."
- Sketch = `'handDrawn'`. Label: "Sketch". Description: "Hand-drawn rough style."

In the **Apply tab**, add a compact version (3 small buttons) in the palette strip area.

### Update Theme Engine

In `themeEngine.ts`, include `look` in the init directive output:

```ts
// When look is 'default', omit (it's the Mermaid default)
// When look is 'neo' or 'handDrawn', include it:
const lookConfig = look !== 'default' ? `"look": "${look}",` : '';

// Output:
// %%{init: { "theme": "base", "look": "neo", "themeVariables": { ... } }}%%
```

### Update ALL Exports

- **Styled Code**: Include `look` in init directive
- **Markdown Bootstrap**: Add `Look: Neo` row
- **Prompt Scaffold**: Instruct LLM to use the specified look
- **Theme JSON** (if export exists): Include `"look": "neo"`

### Update Preview

The Mermaid preview renderer must pass `look` to `mermaid.initialize()`:

```ts
mermaid.initialize({
  theme: 'base',
  look: selectedLook,
  themeVariables: { ... }
});
```

## Step 3.3: Family-Specific themeVariables

Create `src/lib/familyVariables.ts`:

The theme engine currently generates the same themeVariables block for all diagram families. Different families use different variable sets.

```ts
export type PaletteColors = {
  primaryColor: string;
  primaryTextColor: string;
  primaryBorderColor: string;
  secondaryColor: string;
  secondaryTextColor: string;
  secondaryBorderColor: string;
  tertiaryColor: string;
  tertiaryTextColor: string;
  tertiaryBorderColor: string;
  lineColor: string;
  textColor: string;
  edgeLabelBackground: string;
  [key: string]: string;
};

export const FAMILY_VARIABLES: Record<string, (p: PaletteColors) => Record<string, string>> = {

  flowchart: (p) => ({
    primaryColor: p.primaryColor,
    primaryTextColor: p.primaryTextColor,
    primaryBorderColor: p.primaryBorderColor,
    secondaryColor: p.secondaryColor,
    secondaryTextColor: p.secondaryTextColor,
    secondaryBorderColor: p.secondaryBorderColor,
    tertiaryColor: p.tertiaryColor,
    tertiaryTextColor: p.tertiaryTextColor,
    tertiaryBorderColor: p.tertiaryBorderColor,
    lineColor: p.lineColor,
    textColor: p.textColor,
    mainBkg: p.primaryColor,
    nodeBorder: p.primaryBorderColor,
    clusterBkg: p.tertiaryColor,
    clusterBorder: p.tertiaryBorderColor,
    edgeLabelBackground: p.edgeLabelBackground,
    nodeTextColor: p.primaryTextColor,
  }),

  sequence: (p) => ({
    actorBkg: p.primaryColor,
    actorBorder: p.primaryBorderColor,
    actorTextColor: p.primaryTextColor,
    actorLineColor: p.lineColor,
    signalColor: p.lineColor,
    signalTextColor: p.textColor,
    labelBoxBkgColor: p.secondaryColor,
    labelBoxBorderColor: p.secondaryBorderColor,
    labelTextColor: p.secondaryTextColor,
    loopTextColor: p.textColor,
    noteBkgColor: p.tertiaryColor,
    noteBorderColor: p.tertiaryBorderColor,
    noteTextColor: p.tertiaryTextColor,
    activationBkgColor: p.secondaryColor,
    activationBorderColor: p.secondaryBorderColor,
    sequenceNumberColor: p.primaryTextColor,
  }),

  pie: (p) => ({
    pie1: p.primaryColor,
    pie2: p.secondaryColor,
    pie3: p.tertiaryColor,
    pie4: p.primaryBorderColor,
    pie5: p.secondaryBorderColor,
    pie6: p.tertiaryBorderColor,
    pie7: p.lineColor,
    pie8: p.edgeLabelBackground,
    pieTitleTextColor: p.textColor,
    pieSectionTextColor: '#FFFFFF',
    pieLegendTextColor: p.textColor,
    pieStrokeColor: p.lineColor,
    pieStrokeWidth: '2px',
    pieOpacity: '0.85',
  }),

  state: (p) => ({
    labelColor: p.textColor,
    altBackground: p.tertiaryColor,
    compositeBackground: p.secondaryColor,
    compositeBorder: p.secondaryBorderColor,
    compositeTitleBackground: p.primaryColor,
    innerEndBackground: p.primaryColor,
    transitionColor: p.lineColor,
    transitionLabelColor: p.textColor,
    specialStateColor: p.primaryBorderColor,
  }),

  timeline: (p) => ({
    cScale0: p.primaryColor,
    cScale1: p.secondaryColor,
    cScale2: p.tertiaryColor,
    cScale3: p.primaryBorderColor,
    cScale4: p.secondaryBorderColor,
    cScale5: p.tertiaryBorderColor,
    cScaleLabel0: p.primaryTextColor,
    cScaleLabel1: p.secondaryTextColor,
    cScaleLabel2: p.tertiaryTextColor,
  }),

  xychart: (p) => ({
    backgroundColor: 'transparent',
    titleColor: p.textColor,
    xAxisLabelColor: p.textColor,
    xAxisTitleColor: p.textColor,
    xAxisTickColor: p.lineColor,
    xAxisLineColor: p.lineColor,
    yAxisLabelColor: p.textColor,
    yAxisTitleColor: p.textColor,
    yAxisTickColor: p.lineColor,
    yAxisLineColor: p.lineColor,
    plotColorPalette: `${p.primaryColor},${p.secondaryColor},${p.tertiaryColor}`,
  }),
};

export function getVariablesForFamily(palette: PaletteColors, family: string): Record<string, string> {
  const generator = FAMILY_VARIABLES[family] || FAMILY_VARIABLES.flowchart;
  return generator(palette);
}
```
