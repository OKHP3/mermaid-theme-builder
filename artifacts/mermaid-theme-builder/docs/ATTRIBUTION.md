# Attribution

Mermaid Theme Builder supports two optional attribution layers in generated exports.

Both are configurable per session via the **Export Settings** panel in the sidebar.

---

## Layer 1 — Metadata Comments

**Default:** ON  
**Toggle:** "Metadata comments" in Export Settings

When enabled, all generated Mermaid code includes a block of `%%` comment lines immediately after the `%%{init}%%` directive:

```mermaid
%%{init: {"theme": "base", "themeVariables": { ... }}}%%
%% Created with: Mermaid Theme Builder v0.1.0
%% Tool: https://overkillhill.com/projects/mermaid-theme-builder/
%% Theme: OverKill Hill P³
%% Theme ID: overkill-hill
%% Theme Version: 0.1.0
%% Generated: 2026-01-01T00:00:00.000Z
%% Brand source: https://overkillhill.com
%% Not affiliated with Mermaid, Mermaid Chart, or Mermaid.ai
flowchart TD
    ...
```

`%%` comments are part of the Mermaid specification and are silently ignored by all compliant renderers. They do not affect the visual output of the diagram.

### When metadata comments are included

- **Styled Code** export: YES, always
- **Markdown Bootstrap** export: included in the fenced code block
- **Prompt Scaffold** export: included in the required comment block section

---

## Layer 2 — Attribution Badge Node

**Default:** OFF  
**Toggle:** "Attribution badge" in Export Settings  
**Supported diagrams:** Flowchart only

When enabled for flowchart diagrams, a small disconnected badge node is appended to the diagram code:

```mermaid
    _mtb_attr["Themed with Mermaid Theme Builder · OverKill Hill P³"]
    style _mtb_attr fill:#181f26,stroke:#c46a2c,color:#e5e7eb,font-size:9px,opacity:0.7
    click _mtb_attr href "https://overkillhill.com/projects/mermaid-theme-builder/" _blank
```

The badge is styled using the selected theme's secondary color, border color, and text color, so it integrates visually with the diagram.

### Diagram family support

| Family | Badge support |
|---|---|
| flowchart | ✅ Full (styled + clickable link) |
| sequenceDiagram | ✅ Styled node only |
| stateDiagram | ✅ Styled node only |
| classDiagram | ✅ Styled node only |
| All others | ❌ Falls back to metadata comments only |

The app automatically disables the badge toggle for non-supported diagram families.

### Layout note

The badge node is disconnected from the diagram flow. Most Mermaid renderers will place it in an unused area of the layout. However, diagram layout is renderer-dependent — the badge may appear in unexpected positions. It is best-effort and optional. Test in your target renderer before relying on it.

## Markdown Bootstrap Attribution

The **Markdown Bootstrap** export always includes an attribution footer:

```markdown
Generated with [Mermaid Theme Builder](https://overkillhill.com/projects/mermaid-theme-builder/) · Theme: **OverKill Hill P³**

> _Not affiliated with Mermaid, Mermaid Chart, or Mermaid.ai. All transformations are local — your diagram code never leaves the browser._
```

---

## Prompt Scaffold Attribution

The **Prompt Scaffold** export closes with:

```
*Generated by [Mermaid Theme Builder](...) — paste this scaffold into your AI prompt to maintain visual consistency across all diagrams.*
*Not affiliated with Mermaid, Mermaid Chart, or Mermaid.ai.*
```
