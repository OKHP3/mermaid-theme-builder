# Theming Model

> Mermaid Theme Builder is a personal OverKill Hill P³ project by Jamie Hill. It is not affiliated with Builders FirstSource, BFS, Mermaid, Mermaid Chart, Mermaid.ai, or any third-party brand represented by user-entered colors.

This document explains how Mermaid Theme Builder turns a chosen palette into a styled Mermaid diagram.

## Three layers

Mermaid supports three layers of styling. The Theme Builder uses all three.

| Layer | Mechanism | When we use it |
|------|-----------|----------------|
| **1. Init directive** | `%%{init: { "theme": "base", "themeVariables": { ... } } }%%` at the top of a diagram | Always — sets the palette globally for the diagram. |
| **2. classDef** | `classDef brand fill:#... ,stroke:#...` + `class NodeId brand` | Flowchart, class diagram, state diagram, block diagram only. Adds explicit hot-spots for cross-cutting brand colors. |
| **3. linkStyle** | `linkStyle 0 stroke:#... ,stroke-width:2` | Reserved for future use. Currently not emitted by the engine. |

## Palette → init directive

Every palette is a `ThemeColor[]`. Each entry is a `{ key, label, value }` triple:

```ts
{ key: "primaryColor",       label: "Primary background", value: "#1C3A34" },
{ key: "primaryTextColor",   label: "Primary text",       value: "#FFFFFF" },
{ key: "primaryBorderColor", label: "Primary border",     value: "#0F1F1C" },
// ...
{ key: "fontFamily",         label: "Font family",        value: "Trebuchet MS, Calibri, sans-serif" },
```

The engine flattens this into a single object and writes it into the `themeVariables` block of the init directive. `fontFamily` is included inside `themeVariables` (Mermaid v11 honours it there); the entry is intentionally appended last so it is easy to read in the emitted directive.

## Family-aware overlay

Different Mermaid diagram families look at different `themeVariables` keys. A flowchart cares about `nodeBkg`, `clusterBkg`, `edgeLabelBackground`. A sequence diagram cares about `actorBkg`, `actorBorder`, `signalColor`. A pie chart cares about `pie1`–`pie8`.

`familyTheming.ts` produces an **overlay** for each family by mapping the palette's well-known tokens (primary, secondary, tertiary, line, text) to the family-specific keys.

```ts
function buildInitDirective(palette: Palette, family: DiagramFamily) {
  const baseVars  = paletteToVars(palette);            // user-facing tokens
  const overlay   = familyThemeOverlay(palette, family); // family-specific keys
  // Base palette wins: any explicit user value overrides the computed overlay.
  const themeVars = { ...overlay, ...baseVars };
  return formatInit(themeVars, palette.fontFamily);
}
```

The merge order is intentional: if the user explicitly set, e.g., `actorBkg`, that always wins over the computed overlay. The overlay only fills in defaults for family-specific keys the palette did not name.

## classDef integration

`generateThemedCode` does **not** mass-rewrite the user's diagram. It strips any pre-existing `%%{init}%%` block, prepends the new init directive, and (when the badge is enabled and the family supports clickable nodes) appends a small `MTB_BADGE` classDef + node. The user keeps full control of which nodes opt into any brand class.

A larger 16-class semantic `classDef` library is generated by the **scaffold/markdown export** (`buildScaffold` in `themeEngine.ts`) for `CLASSDEF_CAPABLE_FAMILIES` (`flowchart`, `classDiagram`, `stateDiagram`, `block`). It is meant to be pasted into the LLM prompt or a downstream documentation block — not silently injected into the rendered preview. This keeps the live preview faithful to what the user pasted.

## Watermark and metadata

When `includeBadge` is on (default for badge-safe families) and the diagram supports clickable nodes, a small `MTB_BADGE` node is appended with a click-through to the tool URL. When `includeMetaComments` is on, a short `%% Theme: …` header comment is added.

Both can be disabled. Neither emits any analytics or network call.

## Adding a new palette

1. Append a new `Palette` to `BRAND_PALETTES` or `UTILITY_PALETTES` in `lib/palettes.ts`.
2. Provide all 13+ keys: primary/secondary/tertiary triples, plus `lineColor`, `textColor`, `mainBkg`, `clusterBkg`, `clusterBorder`, `edgeLabelBackground`, `fontFamily`.
3. The Brand Firewall (`AGENTS.md`) prohibits real third-party brand palettes (BFS, Walmart, Starbucks, Apple, Microsoft, Target, Home Depot, etc.). Use original generic palettes or palettes from the OverKill Hill / AskJamie / Glee-fully ecosystem.

## Adding a new diagram family

1. Add the family to the `DiagramFamily` union in `data/mermaid-capabilities.ts`.
2. Append a `DiagramCapability` row with detection regex, `supportStatus`, `themeConfidence`, `notationCompliance`, and warnings.
3. If the family has its own `themeVariables`, add a branch to `familyThemeOverlay()` in `lib/familyTheming.ts`.
4. Optionally add an example to `data/example-library.ts`.
